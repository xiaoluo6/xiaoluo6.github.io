  16. 新建+启动容器

    • OPTIONS说明
      • 注意： 有些是一个减号，有些是两个减号
      • –name=”容器新名字”
        • 为容器指定一个名字， 如果不指定，则系统随机分配
          • eg: 启动容器，命名为myUbuntu， 开启伪输入端
            • docker run -it –name=myUbuntu ubuntu /bin/bash
      • -d
        • 后台运行容器，并返回容器ID，也即启动守护式容器(后台运行)
      • -i
        • 以交互模式运行容器, 通常与-t同时使用
      • -t
        • 为容器重新分配一个人伪输入终端, 通常与-i同时使用
        • 也即启动交互式容器(前台有伪终端，等待交互)
        • eg: 打开Ubuntu，并开启伪输入端 ,希望有个交互式shell ,退出终端 exit
          • docker run -it ubuntu /bin/bash
      • -P
        • 随机指定端口映射，大写P
      • -p
        • 指定端口映射，小写p

    2. 启动守护式容器(后台服务器)
      • 在大部分的场景下，我们希望docker的服务是在后台运行的，我们可以通过 -d指定容器的后台运行模式
      • docker run -d 容器名
        • 注意：部分容器是无法后台运行，例如Ubuntu， 可通过 docker ps查看启动情况
          • docker run -d ubuntu
          • docker ps
          • 说明：Docker容器后台运行，就必须有一个前台进程
          • 最佳的解决方案是: 将你要运行的程序以前台进程的形式运行
      • redis前后台启动演示案例
        • 前台交互式启动
          • docker run -it redis:6.0.8
          • 按住ctrl+c 后服务即停止了
        • 后台守护式启动
          • docker run -d redis:6.0.8
    3. 查看容器日志
    4. 查看容器内运行的进程
    5. 查看容器内部细节
    6. 进入正在运行的容器并以命令行交互
      • 方式一：docker exec -it 容器ID /bin/bash
      • 方式二：docker attach 容器ID
      • 案例演示， 用ubuntu演示
      • 两种方式的区别
        • exec
          • 在容器中打开新的重点，并且可以启动新的进程，
          • 使用exit退出，不会导致容器的停止
        • attach
          • 直接进入容器启动命令的终端，不会启动新的进程
          • 用exit退出，会导致容器的停止
      • 推荐使用docker exec命令，因为退出容器终端，不会导致容器的停止，
      • 用之前的redis示例进入试试
        • 进入redis服务

      • 设置获取修改图表实例的配置项以及数据
      • 多次调用setOption方法

    2. resize

      • 重新计算和绘制图表
      • 一般和window对象的resize事件合并使用

      • 绑定或解绑事件处理函数

          • 常见的事件：click、dbclick、mousedown、mouseover、mouseup等

      • 使用代码模拟用户的行为

      • 清空之后可以再次setOption

      • 销毁实例
      • 销毁后实例无法再次被使用

2.1 后台开发 Koa2

  1. Koa2特点
    • 支持async\await
    • 洋葱模型的中间件
  2. 中间件的特点
    • Koa对象通过use方法加入一个中间件
    • 一个中间件就是一个函数
    • 中间件的执行顺序符合洋葱模型
    • 内层中间件能否执行取决于外层中间件的next函数是否调用
    • 调用next函数得到的是Promise对象
